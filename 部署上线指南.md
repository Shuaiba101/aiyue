# i阅 / ireading 网页版 —— 傻瓜式上线 + 绑定 ireading.top

整套部署走 **Vercel（海外，免备案）**。代码根目录是 `web/`。
下面命令除特别说明，都在 **`web/` 目录**里执行。

## 0. 一次性准备（约 2 分钟）

```bash
cd /Users/zhouyinyun/aiyue/web
vercel login          # 浏览器登录 Vercel（没账号就用 GitHub/邮箱注册，免费）
```

## 1. 首次部署，拿到临时网址

```bash
vercel              # 第一次会问几个问题，全部回车默认即可；它会创建项目并给一个 *.vercel.app 预览地址
```

回答提示时：
- `Set up and deploy?` → 回车（Y）
- `Which scope?` → 选你的账号
- `Link to existing project?` → N
- `Project name?` → 回车（默认 `web` 或自定义 `ireading`）
- `In which directory is your code?` → 回车（`./`，因为已经在 web 目录里）
- 其余构建设置 → 全部回车默认（Vercel 会自动识别 Next.js）

部署完会输出一个像 `https://ireading-xxx.vercel.app` 的网址。

## 2. 上传环境变量（DeepSeek / MiMo / Tavily / Supabase）

```bash
bash scripts/deploy-env.sh        # 自动把 .env.local 里的 5 个 key 推到 Vercel 生产环境
vercel --prod                     # 用带 key 的配置重新做一次正式生产部署
```

> 脚本只在运行时读取本地 `.env.local`，密钥不会进 git 仓库。

需要上传的变量（脚本会自动处理，列出来供你核对）：
- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `DEEPSEEK_API_KEY`
- `TAVILY_API_KEY`
- `MIMO_API_KEY`

## 3. 绑定域名 ireading.top

### 3.1 在 Vercel 添加域名
```bash
vercel domains add ireading.top
```
或在网页后台：项目 → **Settings → Domains** → 输入 `ireading.top` → Add。
Vercel 会告诉你要在 DNS 里加哪些记录（以它显示的为准）。一般是：

| 类型 | 主机记录 | 值 |
|------|----------|-----|
| A | `@`（根域）| `76.76.21.21` |
| CNAME | `www` | `cname.vercel-dns.com` |

### 3.2 回 NameSilo 设置 DNS
1. 登录 NameSilo → **Domain Manager** → 点 `ireading.top`。
2. 找到 **DNS / Manage DNS**。
3. 按上面 Vercel 给的记录添加：
   - A 记录：Host 填 `@`（或留空），IPV4 填 Vercel 给的 IP。
   - CNAME 记录：Host 填 `www`，Target 填 `cname.vercel-dns.com`。
4. 保存。DNS 生效一般几分钟到 1 小时。

生效后 Vercel 会**自动签发免费 HTTPS 证书**，`https://ireading.top` 就能打开了。

## 4. 让登录功能在新域名上正常（Supabase）

1. 打开 Supabase 控制台 → 你的项目 → **Authentication → URL Configuration**。
2. **Site URL** 填：`https://ireading.top`
3. **Redirect URLs** 里加上：`https://ireading.top/auth/callback`
4. 保存。

（邮箱确认此前已关闭，所以注册即登录，无需收邮件。）

## 以后怎么更新线上版本？

### 方式 A：GitHub 自动部署（推荐）

代码已托管在 [github.com/Shuaiba101/aiyue](https://github.com/Shuaiba101/aiyue)。

```bash
git add -A
git commit -m "你的改动说明"
git push origin main
```

push 到 `main` 后，Vercel 会自动构建并部署到 `https://ireading.top`。

### 方式 B：手动 CLI 部署

改完代码后，在 `web/` 目录跑一句即可：

```bash
vercel --prod
```

## 6. 连接 GitHub 与 Vercel（一次性）

若 Vercel 项目还是 CLI 创建的、尚未连 Git：

1. 打开 [Vercel 控制台](https://vercel.com) → 项目 **web** → **Settings → Git**
2. **Connect Git Repository** → 选 **Shuaiba101/aiyue**
3. **Settings → General → Root Directory** → 填 **`web`** → Save
4. 环境变量已在 Vercel 里的话不用动；新仓库首次连 Git 后 push 一次 `main` 即触发部署

> Root Directory 必须设为 `web`，否则 Vercel 会在仓库根目录找 Next.js 而构建失败。
